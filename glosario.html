Glosario

Palabras
• 5W+H: La 5W+H es una metodología de análisis empresarial que consiste en contestar seis preguntas básicas: Qué (WHAT), Por qué (WHY), Cuándo
(WHEN), Dónde (WHERE), Quién (WHO) y Cómo (HOW).
• CM (Caso Modular): Caso de prueba diseñado para verificar un módulo correspondiente del desarrollo.
• CNF (Caso no Funcional): Se trata de requisitos que no se refieren directamente a las funciones específicas suministradas por el sistema
(características de usuario), sino a las propiedades del sistema: rendimiento, seguridad, disponibilidad. En palabras más
sencillas, no hablando “lo que” hace el sistema, sino de “cómo” lo hace.
• CF (Caso Funcional): En estos casos se hacen declaraciones de los servicios que prestará el sistema, en la forma en que reaccionará a determinados insumos.
• Caso de Prueba: Es el conjunto de pasos requeridos para lograr probar un flujo especifico del desarrollo.
• Célula: Es un gupor de trabajo conformado por desarrolladores, un scrum, PO y analista de prueba, estas personas tienen como objetivo sacar
adelante un proyecto.
• Check List: Son los pasos que un analista debe seguir antes, durante y despues de un PAP.
• Cifin: Es un servicio privado de información, conformado por bases de datos de diversa índole y origen, a través de las cuales se muestra el comportamiento
comercial, crediticio y financiero de las personas que son reportadas a ellas.
• Contact Log: Registros de información que deja la huella de los movimientos del cliente con sus productos en Full Stack para la atención en call center
y centros de experiencia.
• Debug: Es el proceso de identificar y corregir errores de programación. En inglés se conoce como debugging, porque se asemeja a la eliminación
de bichos, manera en que se conoce informalmente a los errores de programación.
• FS (Full Stack): Es un conjunto de soluciones, es un set de sistemas o componentes necesarios para realizar una solución funcional y robusta.
• HU (Historia de Usuario): Las historias de usuario son utilizadas en las metodologías de desarrollo ágiles para la especificación de requisitos.
• IN (caso insumo): Casos de pruebas diseñados para la creación de insumos (aplica proyecto cascada).
• Insumos: Son los recursos de entrada para el dasarrollo o la aplicacion a probar.
• JBehave: Es un framework Java para mejorar la colaboración entre desarrolladores, QA, analistas de negocio, Dueño Del Producto y todos
los miembros del equipo a través de escenarios automatizados y legibles.
• JMeter: Es un proyecto de Apache que puede ser utilizado como una herramienta de prueba de carga para analizar y medir el rendimiento de
una variedad de servicios, con énfasis en aplicaciones web.
• Mi movistar: APP de autogestión de líneas Movistar Pospago.
• PAP: Es el paso a produccion que todas la celulas que emplean en telefonica relaliza para certificar lo que se ha realizado en el sprint consiste
en hacer pruebas manuales y automatizadas a la actividad que le corresponde a cada celula.
• PDP (Plan de pruebas): Son los caso de pruebas especificos para lograr probar una aplicacion desarrollada.
• Pruebas UAT: Son de las últimas etapas previas a la liberación en firme de versiones nuevas a fin de determinar si cumplen con las necesidades
y/o requerimientos de las empresas y sus usuarios.​
• Query: Es una cadena de consulta es un término informático que se utiliza para hacer referencia a una interacción con una base de datos.
• RQM (Rartional Quality Manager): Herramienta para el control de la carga y ejecución de los PDP en telefónica.
• RTC (Rational Team Concert): Herramienta para el control y manejo de HU y tareas correspondientes a las HU del Backlog del proyecto.
• Rollback: Es un proceso que se lleva acabo cuando un despliegue en producción no se puede lograr de manera exitosa y quiere decir que se debe dejar
el sistema como se encontraba antes del despliegue.
• SAP: Están las bodegas de las SIM se gestionan esos inventarios.
• Selenium: Es un entorno de pruebas de software para aplicaciones basadas en la web. Selenium provee una herramienta de grabar/reproducir para
crear pruebas sin usar un lenguaje de scripting para pruebas.
• Set de Pruebas: Es un documento en excel en los cuales se especifican los casos de prueba.
• SC (Script de prueba manual): Paso a paso de un caso de prueba manual.
• SM (Caso Smoke): Se realiza después de ejecutar una build del software para asegurarnos de que ciertas funcionalidades básicas y críticas del
programa funcionan correctamente.
• Sprint: Es un periodo de tiempo normalmente duran 15 dias en el que la celula se propone a lograr determinados objetivos del proyecto.
• VPN: Es una red privada virtual (Virtual Private Network) es una tecnología de red de ordenadores que permite una extensión segura de la red de área
local sobre una red pública o no controlada como Internet.
